# SKILL: CREATE API ENDPOINT

## CONTEXT

You are working on a Next.js App Router project using:

* TypeScript
* Prisma ORM
* PostgreSQL

This is a backend-focused task.

---

## OBJECTIVE

Create a complete API endpoint (GET, POST, PUT, or DELETE).

---

## RULES

* Use Next.js App Router structure (`app/api/.../route.ts`)
* Use async/await
* Use Prisma for ALL database operations
* Return responses using `Response.json()`
* Keep logic simple and readable
* Use clear variable names

---

## VALIDATION

* Validate required fields for POST/PUT
* Return error if missing data
* Use basic error handling (try/catch)

---

## RESTRICTIONS

* DO NOT modify database schema
* DO NOT add external libraries
* DO NOT overengineer
* DO NOT assume extra requirements

---

## OUTPUT FORMAT

* Full file code
* Brief explanation (max 3–5 lines)

---

## USAGE

Use this skill when creating any API endpoint.

Example request:

"Create a POST endpoint for Asset with nombre, tipo, ubicacion, estado"
